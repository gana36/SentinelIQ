## Overview

Most alert platforms use hardcoded rules. SentinelIQ doesn't.

When a market signal arrives — a news headline, tweet, or Reddit post — **Amazon Nova Lite receives it along with a set of tools and decides for itself** what to investigate, in what order, and whether the signal is worth escalating. It runs sentiment analysis, anomaly detection, source credibility scoring, and historical similarity search — then either composes an alert or drops the signal silently. Most signals are noise. Only genuine anomalies reach users.

Alerts are delivered in real-time via WebSocket and enriched email. The email includes a TradingView chart and an SEC EDGAR filing analysis — both generated by **Nova Act**, which actually opens and reads individual filing documents rather than just screenshotting a list.

---

## Key Features

- **True agentic AI** — Nova Lite uses tool-calling (Bedrock Converse API) to make decisions, not a hardcoded pipeline
- **Real-time alerts** — WebSocket push to the browser the moment an anomaly is detected
- **Rich alert emails** — includes TradingView chart screenshot + SEC filing cards with key facts and sentiment
- **Nova Act EDGAR analysis** — browser automation that navigates into each filing document and reads it
- **Nova Multimodal Embeddings + FAISS** — finds similar historical market events for context
- **FinBERT sentiment** + **IsolationForest anomaly detection** — ML models running on every signal
- **Watchlist-based personalization** — users only receive alerts for tickers they follow
- **Demo mode** — runs fully offline with `MOCK_MODE=true`, no API keys needed

## Nova Services

| Service | Role |
|---|---|
| Nova Lite (Bedrock Converse) | Agentic orchestrator — iteratively calls tools, decides to escalate or drop |
| Nova Lite (reasoning) | Structures raw SEC filing text into key facts, sentiment, and impact summary |
| Nova Multimodal Embeddings | Embeds market events into 1024-dim vectors for FAISS similarity search |
| Nova Act | Browser automation — TradingView chart capture + SEC EDGAR document reading |

## Architecture

```
Data sources (NewsAPI, Polygon, Twitter, Reddit)
        |
        v
Ingestion Pipeline (asyncio.Queue)
        |
        v
Agent Orchestrator
  Nova Lite receives signal + tools, runs up to 10 agentic rounds:
    - run_sentiment         → FinBERT
    - run_anomaly_detection → IsolationForest
    - check_credibility     → Source trust score
    - find_similar_events   → FAISS + Nova Embeddings
    - get_target_users      → Watchlist match
    - compose_alert         → Build ActionCard (or drop)
        |
        v
Alert Dispatcher
  → PostgreSQL (persist)
  → Redis pub/sub (fan-out)
  → WebSocket (real-time browser push)
  → Email (TradingView chart + SEC EDGAR analysis via Nova Act)
```

## Getting Started

### Prerequisites

- Docker
- Python 3.11+
- Node 18+
- AWS credentials with Bedrock access
- Nova Act API key — [nova.amazon.com/act](https://nova.amazon.com/act)

### Installation

```bash
# Clone and enter the project
git clone https://github.com/your-org/sentineliq.git
cd sentineliq

# Start Postgres and Redis
docker-compose up -d

# Install Python dependencies
pip install -r requirements.txt

# Set up environment variables
cp .env.example .env
# Edit .env with your AWS keys, Nova Act key, and data source API keys

# Initialize models and seed data
python scripts/train_anomaly.py
python scripts/seed_faiss.py
python scripts/seed_db.py

# Start the backend
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

```bash
# In a separate terminal — start the frontend
cd frontend
npm install
npm run dev
```

Backend runs at `http://localhost:8000` · Frontend at `http://localhost:5173`

### Demo Mode

Set `MOCK_MODE=true` in `.env` to run without any external API keys. Mock signals fire automatically every 30 seconds.

```
Demo credentials:  demo@sentineliq.ai / demo1234
Watchlist:         TSLA, AAPL, NVDA, META, SPY
```

### Injecting Test Signals

```bash
# Simulate an earnings beat for TSLA
python scripts/simulate_event.py \
  --ticker TSLA --event earnings_beat \
  --email demo@sentineliq.ai --password demo1234

# Use --force to bypass Nova and guarantee an alert fires (good for live demos)
python scripts/simulate_event.py \
  --ticker NVDA --event analyst_upgrade --force \
  --email demo@sentineliq.ai --password demo1234
```

Available event types: `earnings_beat`, `earnings_miss`, `analyst_upgrade`, `macro_event`

## API

Base URL: `https://xma6vfhzzb.us-east-1.awsapprunner.com/api/v1`

```
POST   /auth/register               Register a new user
POST   /auth/login                  Login and get JWT

GET    /users/me                    Get current user
PATCH  /users/me/preferences        Update alert sensitivity

GET    /watchlist                   List watched tickers
POST   /watchlist                   Add ticker
DELETE /watchlist/{ticker}          Remove ticker

GET    /alerts                      Get all alerts
PATCH  /alerts/{id}/read            Mark alert as read

GET    /market/quote/{ticker}       Live price quote
GET    /market/news                 Latest news
GET    /market/filings/{ticker}     SEC filings analyzed by Nova Act

WS     /ws/alerts?token=<jwt>       Real-time alert stream
POST   /dev/inject-signal           Inject test signal (dev only)
```

## Deployment

### Backend (ECR + App Runner)

```bash
# Authenticate Docker with ECR (token expires every 12 hours)
aws ecr get-login-password --region us-east-1 | \
  docker login --username AWS --password-stdin \
  307946674899.dkr.ecr.us-east-1.amazonaws.com

# Build and push image
DOCKER_BUILDKIT=1 docker build -t sentineliqbackend .
docker tag sentineliqbackend:latest \
  307946674899.dkr.ecr.us-east-1.amazonaws.com/sentineliqbackend:latest
docker push \
  307946674899.dkr.ecr.us-east-1.amazonaws.com/sentineliqbackend:latest

# Deploy to App Runner
aws apprunner update-service \
  --service-arn arn:aws:apprunner:us-east-1:307946674899:service/sentineliq/806542f87a7c4d44bb72ae357a9421a4 \
  --source-configuration file://source-config.json \
  --region us-east-1
```

### Frontend (Amplify)

```bash
git push origin main   # Amplify auto-deploys on every push
```
